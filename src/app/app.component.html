Option A — Fix Applied

Remove the *ngIf hover gate

The root fix is to stop hiding the button behind *ngIf="isRowHovered". Instead, keep the button in the DOM at all times and hide it with CSS (opacity: 0).

  • Before (bug): *ngIf removes the button from DOM → @ViewChild is undefined → TypeError.
  • After (fix): button always in DOM → @ViewChild always resolves → EPF click succeeds.
❌ Bug  —  <button *ngIf="isRowHovered">View Details</button>

Option B — Defensive Coding

Guard the @ViewChild read

If the *ngIf hover gate must stay, guard the @ViewChild reference before accessing it. This prevents the unhandled TypeError — but EPF’s click will still silently do nothing, because the button is not in the DOM.

❌ Before — const w = this.viewBtnRef.nativeElement.getBoundingClientRect().width;
⚠️ After — if (this.viewBtnRef) { const w = this.viewBtnRef.nativeElement.getBoundingClientRect().width; }

⚠️ This is not a full fix. The app no longer crashes, but EPF’s click is silently ignored — the modal never opens.
