⚠ Modal / ViewChild Bug Reproduction

Faithful EPF reproduction. Row ORD-003's View Details button is wrapped in *ngIf="isRowHovered" — it only enters the DOM after mouseenter fires. The @ViewChild('viewBtn') reference is therefore undefined until the mouse hovers the row.

Native mouse path: cursor moves over row → mouseenter fires → isRowHovered = true → button renders → click → viewBtnRef exists → ✅ works.
EPF / SenseTalk path: jump-click, no prior hover → mouseenter never fires → isRowHovered = false → button NOT in DOM → viewBtnRef is undefined → ❌ TypeError: Cannot read properties of undefined (reading 'width').

Fixed pattern: always set isModalOpen = true first so Angular renders the element, then read its dimensions inside setTimeout(0) — after change detection has run.
